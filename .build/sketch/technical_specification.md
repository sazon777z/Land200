# Техническое Задание (ТЗ): Проект "LC200 Smart Desktop Watch"

## 1. Общие сведения
Цель проекта — создание настольных смарт-часов в корпусе масштабной модели автомобиля Land Cruiser 200. Устройство должно отображать время и погоду, синхронизируемые через интернет, поддерживать функцию будильника с аудиоэффектами (звуки двигателей) и иметь современный веб-интерфейс для настройки.

---

## 2. Аппаратная конфигурация (Hardware)

### 2.1. Основные компоненты
* **Контроллер:** ESP32-C3 Super Mini.
* **Дисплей:** 2.4" TFT LCD 240x320 (контроллер ILI9341).
* **Аудио:** DFPlayer Mini MP3.
* **Адресная подсветка:** 4x WS2812B (Rainbow эффект).
* **Освещение:** Стандартные светодиоды (Передние фары и Задние фонари).
* **Управление:** 1 тактовая кнопка (Alarm Stop / Mode).

### 2.2. Схема подключения (Pinout Mapping)
| Компонент       | Пин ESP32-C3 | Тип сигнала | Примечание                  |
|-----------------|--------------|-------------|-----------------------------|
| **ILI9341 SCK** | GPIO 6       | SPI CLK     |                             |
| **ILI9341 MOSI** | GPIO 7       | SPI MOSI    |                             |
| **ILI9341 CS** | GPIO 4       | Output      |                             |
| **ILI9341 DC** | GPIO 5       | Output      |                             |
| **ILI9341 RST** | GPIO 8       | Output      |                             |
| **ILI9341 BL** | GPIO 10      | PWM/Output  | Подсветка экрана            |
| **WS2812B DIN** | GPIO 2       | Digital Out | Цепочка из 4-х диодов       |
| **DFPlayer RX** | GPIO 21 (TX) | UART        | Через резистор 1кОм         |
| **DFPlayer TX** | GPIO 20 (RX) | UART        |                             |
| **Front LEDs** | GPIO 0       | Output/PWM  | Передние фары               |
| **Rear LEDs** | GPIO 1       | Output/PWM  | Задние фонари               |
| **Button** | GPIO 3       | Input Pullup| Прерывание по нажатию       |

## 3. Программные требования (Software)

### 3.1. Используемые библиотеки
* **Дисплей:** `Arduino_GFX` (строго согласно условию).
* **WiFi/Network:** `WiFi.h`, `HTTPClient.h`, `NTPClient.h`.
* **LEDs:** `FastLED` или `Adafruit_NeoPixel`.
* **Audio:** `DFRobotDFPlayerMini`.
* **Web Server:** `ESPAsyncWebServer`, `LittleFS` (для хранения UI).

### 3.2. Функциональные модули
1.  **Time & Weather Module:** * Синхронизация времени по протоколу NTP.
    * Получение данных о температуре через OpenWeatherMap API (или аналогичный сервис) по JSON.
2.  **Display Engine:** * Отображение цифровых/аналоговых часов.
    * Вывод температуры и иконок погоды на экран 240x320.
3.  **Alarm System:**
    * Сравнение текущего времени с установленным в энергонезависимой памяти (EEPROM/LittleFS).
    * Активация MP3 трека при срабатывании.
    * Мигание фарами (GPIO 0, 1) в противофазе с частотой 2 Гц.
4.  **Visual Effects:**
    * Режим ожидания: Плавный перелив "Радуга" на WS2812B.
    * Режим будильника: Резкие вспышки красного/белого на WS2812B.

## 4. Веб-интерфейс (Web UI)

### 4.1. Дизайн и UX
* **Стиль:** Dark Mode, стеклянный морфизм (Glassmorphism).
* **Адаптивность:** Корректное отображение на смартфонах и ПК.

### 4.2. Функционал управления
* **Dashboard:** Отображение текущего статуса системы.
* **Time Settings:** Выбор часового пояса (Timezone) и региона для парсинга температуры.
* **Alarm Manager:** * Установка времени будильника.
    * Список доступных звуков двигателей (V8, V6, Diesel и т.д.).
    * Кнопка "Play/Test" для прослушивания выбранного звука перед сохранением.
* **System:** Обновление прошивки, перезагрузка.

---

## 5. Алгоритм работы (Логика)

1.  **Старт:** Инициализация периферии -> Подключение к WiFi (если нет — запуск режима AP для настройки).
2.  **Цикл (Idle):** * Обновление времени каждую секунду.
    * Обновление погоды каждые 15-30 минут.
    * WS2812B вращают цветовой спектр.
3.  **Будильник:**
    * Команда DFPlayer на воспроизведение трека.
    * Запуск цикла мигания фар.
    * Ожидание нажатия физической кнопки или деактивации в Web-интерфейсе.
4.  **Кнопка:** Короткое нажатие при активном будильнике — Stop.

---

## 6. Особые указания
* **Питание:** Учитывая пиковые токи DFPlayer и WS2812B, использовать стабильный источник 5V 2A.
* **Оптимизация:** Использовать FreeRTOS задачи для разделения логики дисплея и сетевых запросов во избежание "фризов" анимации.
